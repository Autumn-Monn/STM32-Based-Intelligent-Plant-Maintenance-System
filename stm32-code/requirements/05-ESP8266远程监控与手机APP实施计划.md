# 阶段10：ESP8266 远程监控与手机 APP — 实施计划

## 1. 阶段目标

在已完成闭环控制系统（阶段9）的基础上，接入 ESP8266 WiFi 模块，通过 MQTT 协议将传感器数据上报至 OneNET 云平台，并开发 UniApp 手机 APP 实现远程数据查看与设备控制，对齐设计要求文档 `01-智能盆栽养护系统设计要求.md` 中的功能要求 1.(4) 远程监控。

## 2. 对齐设计要求

### 2.1 功能要求 1.(4) 远程监控（创新拓展）

| 要求           | 实现方案                                                      |
| -------------- | ------------------------------------------------------------- |
| Wi-Fi 上传数据 | ESP8266 通过 AT 指令连接 WiFi，MQTT 协议上报                  |
| MQTT 协议      | OneNET 物模型 MQTT 接入（mqtts.heclouds.com:1883）            |
| 手机 APP       | UniApp (Vue) 开发 + HBuilderX 打包 APK                        |
| 远程控制       | APP 通过 OneNET HTTP API 下发命令 → MQTT → ESP8266 → STM32 |

### 2.2 数据上报内容

| 属性名        | 类型        | 来源                       | 说明                     |
| ------------- | ----------- | -------------------------- | ------------------------ |
| temperature   | float       | `g_ctrl.temp_raw / 16.0` | 土壤温度（°C）          |
| soil_moisture | int         | `g_ctrl.soil_val`        | 土壤湿度 ADC 原始值      |
| pump_state    | bool        | `g_ctrl.pump_on`         | 水泵状态                 |
| fan_state     | bool        | `g_ctrl.fan_on`          | 风扇状态                 |
| mode          | enum(0/1/2) | `g_ctrl.mode`            | 运行模式：自动/手动/设置 |
| status        | enum(0-4)   | `g_ctrl.status`          | 系统状态                 |

### 2.3 下行命令（远程控制）

| 命令                   | 动作               |
| ---------------------- | ------------------ |
| pump_ctrl = true/false | 手动模式下开关水泵 |
| fan_ctrl = true/false  | 手动模式下开关风扇 |
| mode_set = 0/1         | 切换自动/手动模式  |

## 3. 硬件约束与资源调整

### 3.1 USART1 资源移交（核心变更）

**现状**：USART1（PA9 TX / PA10 RX）被 `debug_uart` 模块占用，用于串口调试日志输出与命令接收。

**变更方案**：USART1 整体移交给 ESP8266（方案 A）。

| 项目            | 变更前                                     | 变更后                     |
| --------------- | ------------------------------------------ | -------------------------- |
| USART1 用途     | debug_uart（调试日志）                     | ESP8266 AT 通信            |
| debug_uart 模块 | 正常工作                                   | 降级为空操作（函数体为空） |
| main.c 调用     | `debug_uart_init()` 保留但不执行实际操作 | 可选：注释掉或保留空调用   |
| debug_uart.c/.h | 保留文件，函数实现改为空体                 | 不删除，便于调试阶段恢复   |

**降级实施**：

- `debug_uart_init()`：清空函数体，不再启用 USART1 中断接收
- `debug_uart_send_string()` 等所有发送函数：清空函数体
- `debug_uart_rx_callback()`：清空函数体
- `main.c` 中 `HAL_UART_RxCpltCallback` 中对 `debug_uart_rx_callback()` 的调用保留（空调用无副作用）

### 3.2 ESP8266 引脚连接

| 功能                             | 引脚 | 说明                                  |
| -------------------------------- | ---- | ------------------------------------- |
| USART1_TX（STM32 → ESP8266 RX） | PA9  | 发送 AT 指令                          |
| USART1_RX（ESP8266 TX → STM32） | PA10 | 接收 AT 响应                          |
| ESP8266_RST                      | PB10 | GPIO 输出，硬件复位控制（低电平复位） |

### 3.3 CubeMX 配置变更

USART1 的 HAL 初始化（`MX_USART1_UART_Init`）保持不变：

- 波特率 115200、8N1、无流控
- 中断优先级可能需要调整（ESP8266 数据量大，建议优先级 1）

新增 PB10 为 GPIO 输出（如未配置）：ESP8266 复位控制。

## 4. ESP8266 AT 指令引擎设计（非阻塞）

### 4.1 设计原则

Demo 代码采用阻塞式 `Delay_ms()` + `ESP_WaitResp()`，不兼容当前非阻塞主循环架构。需改造为**状态机驱动**。

### 4.2 模块文件

| 文件                   | 职责                                        |
| ---------------------- | ------------------------------------------- |
| `User/Src/esp8266.c` | AT 引擎状态机、WiFi/MQTT 连接管理、数据收发 |
| `User/Inc/esp8266.h` | 对外 API 与状态枚举定义                     |

### 4.3 接收缓冲区

```c
#define ESP_RX_BUF_SIZE  512U   // AT 响应可能包含长 JSON，需较大缓冲
static uint8_t g_rx_byte;       // 单字节中断接收
static uint8_t g_rx_buf[ESP_RX_BUF_SIZE];
static volatile uint16_t g_rx_head = 0U;
static volatile uint16_t g_rx_tail = 0U;
```

### 4.4 AT 引擎状态机

```
                    ┌──────────┐
                    │  IDLE    │
                    └────┬─────┘
                         │ esp8266_send_cmd()
                    ┌────▼─────┐
                    │ SENDING  │ ← 通过 HAL_UART_Transmit 发送 AT 指令
                    └────┬─────┘
                         │ 发送完成
                    ┌────▼──────┐
                    │ WAIT_RESP │ ← 在 rx_buf 中搜索期望响应字符串
                    └──┬─────┬──┘
                       │     │
              超时     │     │ 找到匹配
          ┌────────────┘     └────────────┐
     ┌────▼─────┐                   ┌─────▼────┐
     │ TIMEOUT  │                   │   OK     │
     └──────────┘                   └──────────┘
```

### 4.5 WiFi + MQTT 初始化序列（步骤编号驱动）

| 步骤 | AT 指令                                                         | 期望响应        | 超时(ms) | 失败策略                |
| ---- | --------------------------------------------------------------- | --------------- | -------- | ----------------------- |
| 0    | `AT`                                                          | `OK`          | 3000     | 重试3次，失败则硬件复位 |
| 1    | `AT+RST`                                                      | `ready`       | 5000     | 等待模块重启完成        |
| 2    | `AT+CWMODE=1`                                                 | `OK`          | 2000     | —                      |
| 3    | `AT+CWJAP="SSID","PASSWORD"`                                  | `WIFI GOT IP` | 15000    | 重试2次                 |
| 4    | `AT+MQTTUSERCFG=0,1,...`                                      | `OK`          | 6000     | —                      |
| 5    | `AT+MQTTCONN=0,"mqtts.heclouds.com",1883,1`                   | `OK`          | 10000    | 重试2次                 |
| 6    | `AT+MQTTSUB=0,"$sys/{pid}/{dev}/thing/property/set",0`        | `OK`          | 6000     | —                      |
| 7    | `AT+MQTTSUB=0,"$sys/{pid}/{dev}/thing/property/post/reply",0` | `OK`          | 6000     | —                      |

> WiFi SSID/密码与 OneNET 认证参数通过宏定义配置，便于修改。

### 4.6 数据上报流程

```
esp8266_task() 每 30 秒触发一次上报：
  1. 构造 JSON 字符串：
     {"id":"123","params":{"temperature":{"value":25.5},"soil_moisture":{"value":2100},...}}
  2. 计算 JSON 长度
  3. 发送 AT+MQTTPUBRAW=0,"topic",<len>,0,0
  4. 等待 ">" 提示符
  5. 发送 JSON 数据
  6. 等待 "OK" 确认
```

### 4.7 下行命令接收

```
USART1 RX 中断持续接收数据 → 环形缓冲区
esp8266_task() 中定期扫描缓冲区：
  - 检测 "+MQTTSUBRECV:" 前缀
  - 解析 topic 和 payload
  - 匹配 "property/set" topic
  - 解析 JSON 中的命令字段
  - 调用 control 模块对应接口执行动作
```

### 4.8 对外 API

```c
void     esp8266_init(void);          // 初始化引脚、启动连接序列
void     esp8266_task(void);          // 主循环调用，驱动状态机
uint8_t  esp8266_is_connected(void);  // 查询 MQTT 连接状态
void     esp8266_rx_callback(void);   // USART1 RX 中断回调
void     esp8266_publish_data(void);  // 手动触发一次数据上报（可选）
```

### 4.9 连接状态管理

```
ESP_CONN_IDLE → ESP_CONN_WIFI_CONNECTING → ESP_CONN_WIFI_OK
  → ESP_CONN_MQTT_CONNECTING → ESP_CONN_MQTT_OK → ESP_CONN_RUNNING
  ↓（断线检测）
ESP_CONN_DISCONNECTED → 自动重连
```

OLED 显示可增加 WiFi 连接状态图标（如 `W:OK` / `W:--`）。

### 6.4 OneNET HTTP API 封装（核心代码）

```javascript
// utils/onenet-api.js

const BASE_URL = 'https://iot-api.heclouds.com/thingmodel';
const PRODUCT_ID = 'xxxxxxxx';     // 替换为实际产品 ID
const DEVICE_NAME = 'plant-01';    // 替换为实际设备名
const TOKEN = 'xxxxxxxx';          // 替换为实际 token

// 查询设备最新属性
export function getDeviceProperties() {
  return uni.request({
    url: `${BASE_URL}/query-device-property`,
    method: 'GET',
    header: { 'authorization': TOKEN },
    data: {
      product_id: PRODUCT_ID,
      device_name: DEVICE_NAME,
    }
  });
}

// 下发属性设置命令
export function setDeviceProperty(params) {
  return uni.request({
    url: `${BASE_URL}/set-device-property`,
    method: 'POST',
    header: {
      'authorization': TOKEN,
      'Content-Type': 'application/json'
    },
    data: {
      product_id: PRODUCT_ID,
      device_name: DEVICE_NAME,
      params: params   // e.g. { "pump_state": true }
    }
  });
}
```

## 7. 分阶段实施计划

### 阶段 10A：ESP8266 WiFi 连接（STM32 侧）

**目标**：ESP8266 通过 AT 指令成功连接 WiFi 路由器。

| 任务                | 详细说明                                     |
| ------------------- | -------------------------------------------- |
| 1. debug_uart 降级  | 所有函数体清空，保留文件                     |
| 2. 创建 esp8266.c/h | 模块骨架、RX 缓冲区、中断回调                |
| 3. AT 引擎状态机    | 发送-等待-超时 基础框架                      |
| 4. WiFi 连接序列    | AT → RST → CWMODE → CWJAP，非阻塞         |
| 5. 硬件复位逻辑     | PB10 控制 ESP8266 RST 引脚                   |
| 6. OLED 状态显示    | 显示 WiFi 连接状态                           |
| 7. main.c 集成      | 添加 `esp8266_init()` + `esp8266_task()` |

**验收标准**：

- [ ] ESP8266 上电后自动连接指定 WiFi
- [ ] 连接过程不阻塞主循环（其他功能正常运行）
- [ ] OLED 显示 WiFi 连接状态
- [ ] 连接失败可自动重试
- [ ] 编译通过，可烧录运行

### 阶段 10B：OneNET MQTT 连接与数据上报

**目标**：设备成功连接 OneNET MQTT 服务器，周期性上报传感器数据。

| 任务               | 详细说明                           |
| ------------------ | ---------------------------------- |
| 1. MQTT 连接序列   | MQTTUSERCFG → MQTTCONN → MQTTSUB |
| 2. 数据上报        | 每 30 秒构造 JSON 并 MQTTPUBRAW    |
| 3. OneNET 平台配置 | 创建产品/设备/物模型（手动）       |
| 4. 断线重连        | MQTT 断开后自动重新初始化          |
| 5. OLED 状态更新   | 显示 MQTT 连接状态                 |

**验收标准**：

- [ ] OneNET 控制台能看到设备在线
- [ ] 温度、湿度等数据在 OneNET 平台实时更新
- [ ] 上报过程不阻塞主循环
- [ ] 断网后能自动重连

### 阶段 10C：手机 APP 开发（可与 10A/10B 并行）

**目标**：UniApp 手机 APP 能通过 OneNET API 查看设备数据并下发控制命令。

| 任务               | 详细说明                       |
| ------------------ | ------------------------------ |
| 1. UniApp 项目搭建 | CLI 创建项目，VS Code 开发环境 |
| 2. 主页开发        | 数据展示 + 控制按钮            |
| 3. OneNET API 封装 | 查询属性 + 下发命令            |
| 4. 定时刷新        | 每 5-10 秒轮询最新数据         |
| 5. APK 打包        | HBuilderX 云打包               |

**验收标准**：

- [ ] APP 能显示温度、湿度、执行器状态
- [ ] APP 能远程开关水泵、风扇
- [ ] APP 能切换自动/手动模式
- [ ] APK 可安装到 Android 手机

**前置说明**：APP 开发不依赖 STM32 固件完成。可先在 OneNET 控制台手动推送测试数据，APP 对着测试数据开发调试。

### 阶段 10D：下行命令接收与全链路联调

**目标**：APP ↔ OneNET ↔ ESP8266 ↔ STM32 全链路打通。

| 任务             | 详细说明                                                           |
| ---------------- | ------------------------------------------------------------------ |
| 1. MQTT 下行解析 | 解析 `+MQTTSUBRECV:` 中的 JSON 命令                              |
| 2. 命令执行      | 调用 control 模块接口执行水泵/风扇/模式切换                        |
| 3. 状态回报      | 执行完成后立即上报最新状态                                         |
| 4. 全链路测试    | APP 控制 → OneNET → ESP8266 → STM32 → 执行 → 上报 → APP 刷新 |
| 5. 异常处理      | WiFi 断开、MQTT 超时、非法命令等                                   |

**验收标准**：

- [ ] APP 点击"开水泵" → STM32 水泵继电器动作 → APP 状态更新
- [ ] 手动模式下远程控制有效，自动模式下远程控制被拒绝（或提示切换模式）
- [ ] 全链路延迟 < 5 秒
- [ ] 异常情况有合理处理

## 10. 风险与注意事项

| 风险项              | 说明                            | 缓解措施                                           |
| ------------------- | ------------------------------- | -------------------------------------------------- |
| USART1 调试能力丢失 | 接入 ESP8266 后无法串口打印调试 | OLED 显示调试信息；需要时断开 ESP8266 恢复调试     |
| ESP8266 供电不稳    | WiFi 发射时瞬时电流 300mA+      | 确保 3.3V 电源有足够电容（≥100μF），建议独立 LDO |
| AT 响应解析出错     | 多行响应、异步推送混杂          | 环形缓冲区 + 行解析 + 状态机严格匹配               |
| OneNET token 过期   | 认证 token 有有效期             | 计算时设置较长过期时间（如 2030 年）               |
| RX 缓冲区溢出       | MQTT 下发数据量大时             | 512 字节缓冲 + 及时消费 + 溢出保护                 |
| 继电器电路缺陷      | PA1/PA2 继电器存在 PCB 设计缺陷 | 代码逻辑完整保留，逻辑状态正常联动                 |
