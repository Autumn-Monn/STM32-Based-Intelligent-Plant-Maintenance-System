# 阶段09：联动控制逻辑

## 本阶段目标

开始把湿度、温度、继电器、蜂鸣器和指示灯串起来，形成最小自动控制规则；在当前仓库中，这一阶段已经完成了预研代码和接口设计，但尚未正式接入主工程主循环。

## CubeMX 配置

- 本阶段没有新增复杂外设配置，主要复用前面阶段已经完成的资源：
  - `ADC1 / PA0`：土壤湿度输入
  - `PA4`：DS18B20 单总线输入
  - `PA1`：水泵继电器
  - `PA2`：风扇继电器
  - `PA3`：蜂鸣器
  - `PA7 / PB0 / PB1`：状态指示灯
  - `USART1`：模式切换与状态观察
- 当前阶段的联动验证代码位于 `Test/`，因此没有额外修改 CubeMX 生成文件

## AI 指令

- 先实现最小自动控制规则，不急于做完整产品逻辑
- 自动模式至少包含三类行为：
  - 土壤过干时启动水泵
  - 温度过高时启动风扇
  - 异常或超时条件触发蜂鸣器报警与红灯提示
- 同时保留手动模式，便于继电器和报警链路单独验证
- 将规则逻辑抽离为 `control` 模块，避免继续膨胀 `main.c`
- 在真正接入主工程前，先放在 `Test/` 做预研与演示

## 编写逻辑

- `control.c` 中已经预设了自动控制框架：
  - 基于土壤 ADC 原始值判断“干/湿”状态
  - 基于 DS18B20 原始值判断“高温/恢复”状态
  - 水泵带最大连续运行时间保护
  - 设备异常或保护触发时拉起报警状态
