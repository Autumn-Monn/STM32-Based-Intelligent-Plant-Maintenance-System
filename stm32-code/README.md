# STM32 智能盆栽养护系统

基于 **STM32F103C8T6** 的智能盆栽养护系统固件，实时监测土壤湿度与环境温度，自动控制水泵浇灌和风扇散热，并通过 ESP8266 WiFi 模块接入 OneNET 物联网平台实现远程监控。

## 功能特性

- **双传感器采集**：ADC 土壤湿度传感器 + DS18B20 数字温度传感器
- **自动闭环控制**：基于阈值的水泵/风扇自动启停，带滞回消抖防止继电器频繁动作
- **三种运行模式**：自动模式（阈值控制）/ 手动模式（按键控制）/ 设置模式（阈值调整）
- **植物类型预设**：耐旱型 / 常规型 / 喜湿型，三组自适应阈值可逐项微调
- **OLED 实时显示**：128×64 SSD1306 显示温湿度、执行器状态、运行模式与连接状态
- **WiFi 远程监控**：ESP8266 接入 OneNET MQTT，每 30 秒上报数据，支持下行命令控制
- **掉电保存**：AT24C02 EEPROM 存储阈值配置，上电自动恢复
- **多级报警**：三色 LED 分级指示（绿=正常 / 红=缺水 / 蓝=高温）+ 蜂鸣器间歇报警 + K4 静音

## 构建与烧录

### 环境要求

- `arm-none-eabi-gcc`（已加入 PATH）
- CMake ≥ 3.22
- Ninja

### 编译

```bash
# 配置（首次或修改 CMakeLists.txt 后）
cmake --preset Debug

# 编译
cmake --build build/Debug
```

产物位于 `build/Debug/`：`stm32-code.elf`、`.hex`、`.bin`。

### 烧录

```bash
# 方式一：OpenOCD + GDB
openocd -f interface/stlink.cfg -f target/stm32f1x.cfg
# GDB 中执行：target remote :3333; load; continue

# 方式二：st-flash 直接烧录
st-flash write build/Debug/stm32-code.bin 0x08000000
```

## 控制逻辑

### 运行模式

- **自动模式**：根据阈值自动控制——土壤湿度 < 下限时启动水泵，温度 > 上限时启动风扇
- **手动模式**：K2 切换水泵、K3 切换风扇，手动开关执行器
- **设置模式**：K1 长按进入，K2/K3 调整值，K1 短按切换项目，K4 保存退出

### 阈值配置（5 参数）

```c
typedef struct {
  uint8_t  plant_type;   // 植物类型：0=耐旱 1=常规 2=喜湿
  uint16_t soil_low;     // 湿度下限（低于此值开水泵）
  uint16_t soil_high;    // 湿度上限（高于此值关水泵）
  uint8_t  temp_low;     // 温度下限
  uint8_t  temp_high;    // 温度上限（高于此值开风扇）
} threshold_config_t;
```

### 状态指示

| 状态   | LED      | 蜂鸣器   | 含义                   |
| ------ | -------- | -------- | ---------------------- |
| NORMAL | 绿灯常亮 | 静音     | 传感器数据在正常范围内 |
| DRY    | 红灯常亮 | 间歇报警 | 土壤湿度低于下限       |
| HOT    | 蓝灯常亮 | 间歇报警 | 温度高于上限           |

## WiFi 与云平台

系统通过 ESP8266 连接 WiFi 并接入 [OneNET](https://open.iot.10086.cn/) 物联网平台。

**上报属性**（每 30 秒）：温度、土壤湿度、水泵状态、风扇状态、运行模式、系统状态

**下行控制**：通过 OneNET 控制台或 API 下发属性设置命令，固件自动解析执行并回复确认

**连接状态**：OLED 右上角显示 `M:OK`（MQTT 已连接）/ `W:..`（WiFi 连接中）/ `W:Er`（错误）/ `W:--`（未连接）

配置项位于 `User/Inc/esp8266.h`：

```c
#define ESP_WIFI_SSID      "HONORMagic8"
#define ESP_WIFI_PASS      "99999999"
#define MQTT_BROKER        "mqtts.heclouds.com"
#define MQTT_PORT          1883
```

## 设计原则

- **非阻塞架构**：所有定时操作使用 `HAL_GetTick()` + 时间戳比较，主循环中禁止 `HAL_Delay()`
- **中断最小化**：ISR 中仅置标志位和记录时间戳，业务处理在主循环 `_task()` 中完成
- **模块化设计**：每个模块提供 `_init()` + `_task()` 接口，模块间低耦合
- **CubeMX 兼容**：`Core/` 层仅在 `USER CODE` 块内编辑，支持 CubeMX 重新生成
- **渐进式验证**：每个阶段独立可编译、可烧录、有可观察现象、有实施记录归档

## 常见问题

<details>
<summary>ESP8266 无法连接 WiFi</summary>

1. 检查 `esp8266.h` 中的 SSID 和密码
2. 确认 ESP8266 供电充足（峰值电流需 500mA）
3. 检查 USART1 TX/RX 接线（PA9→RX、PA10→TX 交叉连接）
4. 通过 `esp8266_get_state()` 查看状态机停在哪一步

</details>

<details>
<summary>OLED 显示异常</summary>

1. 确认 I2C 地址为 0x78（SSD1306 默认）
2. 检查 PB6/PB7 接线及 4.7kΩ 上拉电阻
3. 尝试 `oled_clear()` + `oled_refresh()` 重置

</details>

<details>
<summary>DS18B20 读取超时</summary>

1. 确认 PA4 接线正确
2. 检查数据线上的 4.7kΩ 上拉电阻（接 3.3V）
3. 缩短连接线长度（单总线对噪声敏感）

</details>

<details>
<summary>土壤湿度读数不稳定</summary>

1. 在传感器输出端并联 0.1µF 电容滤波
2. 增加采样次数：`soil_read_avg(10)`
3. 检查探针是否接触良好

</details>
